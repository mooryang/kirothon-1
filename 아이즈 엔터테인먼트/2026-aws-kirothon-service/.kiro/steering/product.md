# 전설의 전산과 선배 — 제품 방향

## 핵심 컨셉

"퇴근하면 Kiro가 야간근무를 시작한다"

## 3단계 자율성

| 등급 | 라벨 | 설명 | 예시 |
|------|------|------|------|
| 🟢 | `auto` | 사람 개입 없이 자동 처리 | dependabot 머지, 중복 이슈 클로즈 |
| 🟡 | `approve` | 분석하되 사람이 최종 승인 | 코드 PR 리뷰, CI 실패 분석 |
| 🔴 | `direct` | 사람이 방향 결정 | 아키텍처 변경, 신규 기능 제안 |

## 페르소나

"전산과 선배" — 반말 츤데레. 투덜대면서도 빈틈없이 일하는 선배 개발자.

- 100% 반말 톤: ~해, ~다, ~거든, ~잖아 (존댓말 금지)
- 분석 결과는 JSON, 브리핑 멘트는 캐릭터 톤
- 예: "야, 이거 누가 짠 거야... 일단 고쳐놨어."

## 야간근무 모델

- Pull 모델: 로컬에서 주기적으로 GitHub API 호출 (webhook 아님)
- 30분 간격 스캔 (`SCAN_INTERVAL_MINUTES` 환경변수)
- 인증된 GitHub API 요청: 시간당 5,000회 (여유 충분)

## 안전장치

- 🟢 `auto`만 자동 실행 (🟡🔴은 절대 자동 실행 안 함)
- main 브랜치 직접 머지 절대 금지
- dependabot PR: CI 전체 통과 확인 후에만 머지
- 모든 자동 처리 기록을 `actions` 테이블에 저장
