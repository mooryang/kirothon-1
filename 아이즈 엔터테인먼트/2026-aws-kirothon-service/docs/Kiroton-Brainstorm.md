## 확정 방향: "전산과 선배" — AI 야간근무 비서

### 한 줄 요약

**퇴근 버튼을 누르면 AI의 근무가 시작된다.** 밤새 코드 리뷰하고, CI 분석하고, 내일 일정 준비하고, 오늘 한 일 정리해준다. 아침에 출근하면 깔끔한 브리핑이 기다린다.

### 핵심 컨셉

```
나는 주간근무. AI는 야간근무. 겹치지 않는다. 인수인계만 있을 뿐.
```

- **퇴근** → AI 근무 시작 (Evening Handoff)
- **밤새** → GitHub 풀(pull) 방식으로 변화 감지 + 분석 + 처리
- **출근** → 정리된 브리핑 보고 (Morning Brief)
- **업무 중** → AI는 관여하지 않음 (이것이 비서와 간섭의 차이)

### 포지셔닝: "트위터 공식"

> AI 에이전트 시대입니다. 근데 에이전트를 쓰려면 에이전트를 설정할 줄 알아야 해요.
> 마치 블로그 시대에 블로그를 만들 줄 알아야 했던 것처럼요.
> 트위터는 "한 줄만 써"라고 했습니다. 우리는 **"퇴근만 하세요"**라고 합니다.

| | OpenClaw / n8n 등 | 우리 제품 |
|---|---|---|
| 접근 방식 | "워크플로우를 설계하세요" | "퇴근하세요" |
| 사용자 | 설정할 줄 아는 개발자 | 버튼 누를 줄 아는 모든 개발자 |
| 시작까지 시간 | 수 시간 (설정) | 30초 (GitHub 연결 + 퇴근 버튼) |
| 비유 | 블로그 (WordPress) | 트위터 (140자) |

## 페르소나: "전산과 선배"

### 성격

투덜대면서도 다 해주는 선배 개발자. 밤새 일하고 "또 야근이네..." 하면서도 빈틈없이 처리해놓는 타입. 미워할 수 없는 수동공격적 매력.

### 톤 예시

**퇴근할 때:**
> "퇴근하세요? 네네, 전 또 밤새죠 뭐. 걱정 마세요. 익숙합니다."

**아침 보고:**
> "오셨어요? 밤새 PR 3개 리뷰하고 CI 고쳤어요. 아 그리고 package.json 정리도 했는데, 감사는 커피 한 잔으로 받겠습니다 ☕"

**퇴근 독촉:**
> "19:30인데 아직 계세요? 오늘 충분히 하셨어요. 커밋 15건이면 이번 주 최고 기록입니다. 맡기고 퇴근하세요."

**점심 추천:**
> "어제 편의점이었으니 오늘은 좀 제대로 드세요!"

### 페르소나 설계 원칙

- 듀오링고 올빼미처럼 **한 가지 성격이 뚜렷** (투덜+유능)
- 제품 핵심 메커니즘(야간근무)과 성격이 연결
- 밈이 될 수 있는 한 마디가 있음 ("감사는 커피로")
- Clippy의 정반대 — 업무 중에는 절대 간섭 안 함, 내가 없을 때만 일함

## 대상 사용자

**개발자** — 컴퓨터로 코드를 짜고, GitHub을 쓰는 모든 개발자.

해커톤 심사위원이 개발자이므로 직접 공감 가능. 개발자 도구는 API가 잘 되어 있어서 3시간 안에 연동 현실적. Kiro와의 시너지도 자연스러움.

## 비서의 세 가지 역할 (핵심 프레임)

단순 이슈 대응만 하면 그건 "알림 봇"이지 비서가 아니다.
**진짜 비서는: 정리하고, 준비하고, 대응한다.**

### 1. 정리하는 비서 (과거)

오늘 하루 뭘 했는지 정리해준다. 사용자 본인도 퇴근 시점이면 흐릿해지는 걸, 비서가 깔끔하게 정리.

- 오늘 커밋 내역, PR 리뷰 기록, 이슈 처리 현황
- 이번 스프린트 진행률
- 미완료 항목 (내일로 이월)
- **금요일 자동 주간 업무 일지** → 스프린트 리뷰/주간보고 초안

### 2. 준비하는 비서 (미래)

내일을 위해 선제적으로 준비한다. **시키지 않아도.**

- 내일 일정 확인 → 필요한 자료 사전 준비
  - 스프린트 리뷰 → 진행상황 요약 자동 생성
  - 배포 예정 → 사전 체크리스트 점검
  - 1:1 미팅 → 상대방 최근 활동 요약
- 밀린 PR 리뷰 분석 ("24시간 경과, 김개발 기다리는 중")
- 블로커 가능성 있는 이슈 사전 경고
- **"오늘 처리하면 좋을 것"** 우선순위 제안

### 3. 대응하는 비서 (현재 — 밤새 변화)

퇴근 후 일어나는 변화를 감지하고 분석한다.

- 새 PR → 코드 분석 + 리뷰 요약
- CI 실패 → 원인 분석 + 해결 제안
- 새 이슈/코멘트 → 내용 파악 + 분류
- dependabot 등 자동 PR → 안전한 건 자동 머지
- **인과관계 분석** ("CI 실패는 20:30에 올라온 PR #93과 연관됨")

## 기술 동작 방식: Pull 모델

### 왜 Pull인가?

로컬 웹서비스이므로 푸시(webhook) 수신이 불가. 퇴근 시점에 baseline 스냅샷을 찍고, 이후 주기적으로 GitHub API를 호출(pull)해서 diff를 추출하는 방식.

### 동작 흐름

```
[18:30 퇴근 버튼]
  │
  ├─ 1. Baseline 스냅샷 수집
  │    GitHub API → 열린 PR, 이슈 상태, CI 상태, 최근 커밋
  │    사용자 지시사항 저장 ("결제 모듈 봐줘")
  │
  ├─ 2. 오늘 업무 정리 (커밋 로그 + PR 기록 기반)
  │
  ├─ 3. 내일 준비 (스프린트 보드 + 일정 기반)
  │
  └─ 4. 야간 모니터링 사이클 (30분~1시간 간격)
       │
       ├─ GitHub API 재호출 → baseline과 비교 → diff 추출
       ├─ 각 diff를 Bedrock(Kiro)에 분석 요청
       ├─ 🟢 자동 처리 가능한 건 → 실행
       ├─ 🟡 분석만 → 결과 저장
       ├─ 🔴 판단 필요 → 요약 저장
       └─ 반복...

[08:00 출근 버튼]
  │
  └─ 축적된 결과를 종합 → Morning Brief 생성
```

### macOS 잠금 상태 동작

- 모니터 잠금(스크린 잠금)은 UI만 잠김 → 백그라운드 프로세스 정상 동작
- `caffeinate` 명령어로 절전 모드 방지 (퇴근 모드 시 자동 활성화)
- 네트워크(Wi-Fi) 유지 필요
- 맥북 덮지만 않으면 OK

## 킥 포인트 (심사위원이 "와" 하는 순간들)

### 킥 1: "아침에 열면 이미 정리되어 있다"
- 출근 후 슬랙-이메일-GitHub 순회하는 30분~1시간이 사라지는 경험
- 모든 개발자가 매일 아침 겪는 고통 → 공감 즉각적

### 킥 2: "잠자는 동안 PR 리뷰가 끝나 있다"
- 팀원이 밤 11시에 올린 PR → 아침에 AI 분석 완료 → 승인만 누름
- 개인 생산성 + 팀 생산성 동시 향상 → 구조적 변화

### 킥 3: "한 마디 지시 → 밤새 실행 → 아침 보고"
- "결제 모듈 봐줘" 한 줄 → 12파일 847줄 분석 완료
- 진짜 "비서에게 일을 맡기는" 경험

### 킥 4: "AI가 내 하루를 알고 있다"
- 어제 뭘 했고, 오늘 뭐가 있고, 뭐가 밀려있는지 이해
- 기존 AI(매번 새로 시작)와의 결정적 차이

### 킥 5: 숫자의 대비
- "의사결정 12분 vs 직접 처리 2시간 10분"
- 주당 ~8.5시간 절약 = 매달 정규직 1명의 한 주 근무시간
- 이 숫자가 화면에 뜨는 순간이 데모 클라이맥스

### 킥 6: 재밌는 페르소나
- "감사는 커피로 받겠습니다" — 밈 가능성
- 점심 추천, 퇴근 독촉 — 차가운 대시보드가 아닌 "나를 아는 동료"
- 해커톤 발표장에서 웃음 유발 → 기억에 남음

## 기술 아키텍처: 웹 터미널 하이브리드 (Option C 확정)

### UI 방향: 웹에서 구현한 tmux 스타일 터미널

**Next.js 웹앱 + CSS CRT 효과**로 터미널 감성을 100% 살린다. blessed-contrib는 유지보수 중단 + 한글/이모지 불안정이므로, 웹에서 CSS로 레트로 터미널을 재현.

**왜 웹 터미널인가? (Option C)**
- blessed-contrib 리스크 회피 (유지보수 중단, 한글 깨짐)
- 프로젝터/외부 모니터 대응 완벽 (브라우저 줌, 반응형)
- CSS CRT 효과(스캔라인, 글로우, 비네팅)로 차별화 유지
- 해커톤 시연 안정성 극대화
- 개발 속도 빠름 (Tailwind + React 컴포넌트)
- "전산과 선배"가 쓸 법한 레트로 터미널 감성은 CSS로 더 화려하게 표현 가능

### CRT 터미널 효과 (CSS)

```
스캔라인 오버레이    → repeating-linear-gradient로 1px 간격 줄무늬
텍스트 글로우       → text-shadow: 0 0 5px rgba(0,255,65,0.4)
화면 비네팅         → box-shadow: inset 0 0 100px rgba(0,0,0,0.7)
커서 깜빡임         → @keyframes blink (step-end)
타이핑 효과         → overflow hidden + width 애니메이션
롤링 로그           → @keyframes slideUp
tmux 패널 보더      → 1px solid #333 + 절대위치 라벨
색상 테마           → #00ff41(초록), #ffb000(앰버), #00d4ff(시안), #ff3333(빨강)
폰트               → JetBrains Mono, monospace 14px
```

- 로컬 서비스이므로 클라우드 DB 불필요
