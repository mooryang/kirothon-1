# 전설의 전산과 선배 — 컨셉기획서

> "퇴근 버튼을 누르면 Kiro의 근무가 시작된다."

---

## 1. 제품 비전

개발자는 퇴근 후에도 GitHub 알림에 시달린다. 밤새 올라오는 PR, 터지는 CI, 쌓이는 이슈. 아침에 출근하면 어젯밤에 무슨 일이 있었는지 파악하는 것만으로 한 시간이 날아간다.

**전설의 전산과 선배**는 이 문제를 해결한다.

퇴근할 때 "밤새 특별히 봐줄 것"을 한 줄 남기고 나가면, Kiro가 야간 근무를 시작한다. 밤새 GitHub을 모니터링하고, PR을 리뷰하고, CI 실패 원인을 분석하고, 중복 이슈를 정리한다. 아침에 출근하면 레트로 터미널 스타일 대시보드로 야간 브리핑이 기다리고 있다.

개발자는 커피 한 잔 들고 키보드 몇 번 눌러서 승인/거부만 하면 된다.

---

## 2. 핵심 컨셉

### 야간근무 교대 모델

```
낮  → 개발자가 일한다
밤  → Kiro가 일한다
아침 → 개발자가 결과를 검토하고 승인한다
```

사람과 AI가 **교대 근무**를 한다는 개념이다. AI가 사람을 대체하는 게 아니라, 사람이 자는 동안 AI가 야간 당직을 서는 것이다.

### 3단계 자율성 모델

Kiro는 모든 걸 혼자 처리하지 않는다. 상황에 따라 세 가지 레벨로 행동한다.

| 레벨         | 의미                          | 예시                                        |
| ------------ | ----------------------------- | ------------------------------------------- |
| 🟢 자동 처리 | 사람 개입 없이 바로 처리      | dependabot 보안 패치 머지, 중복 이슈 클로즈 |
| 🟡 승인 대기 | 분석은 완료, 사람이 최종 결정 | 코드 PR 리뷰, CI 실패 수정 제안             |
| 🔴 방향 결정 | 사람이 방향을 잡아야 함       | 아키텍처 변경, 신규 기능 방향               |

자잘한 건 알아서 처리하고, 중요한 건 판단 근거를 준비해서 기다린다.

### Pull 모델 (Push 없음)

서버에 웹훅을 설치하거나 GitHub App을 등록할 필요 없다. 로컬에서 30분마다 GitHub API를 호출해서 변화를 감지한다. 설치가 간단하고, 어떤 레포에도 적용할 수 있다.

---

## 3. 타겟 사용자

### 주요 타겟: 1-5인 개발팀 리드

- 혼자 또는 소수 팀으로 여러 레포를 관리하는 개발자
- PR 리뷰, CI 모니터링, 이슈 트리아지를 혼자 다 하는 사람
- 퇴근 후에도 GitHub 알림이 신경 쓰이는 사람
- 아침에 "어젯밤에 뭔 일 있었어?"를 매일 확인해야 하는 사람

### 부차 타겟: 해커톤/사이드 프로젝트 개발자

- 주말에만 작업하는 사이드 프로젝트 관리자
- 컨트리뷰터가 다른 시간대에 있는 오픈소스 메인테이너

### 타겟이 아닌 사람

- 대규모 팀에서 전담 DevOps/SRE가 있는 경우
- 24시간 모니터링 시스템이 이미 갖춰진 조직

## 4. 페르소나: 전산과 선배

제품의 얼굴이자 목소리. Kiro가 이 캐릭터로 말한다.

### 캐릭터 설정

**전산과 선배**는 투덜대면서도 빈틈없이 일하는 선배 개발자다.

- 반말 츤데레: 불평은 하지만 결국 다 해준다
- 수치 먼저, 멘트 나중: "밤새 6건 처리했다. 커피는 사 오는 거지? ☕"
- 기술 용어는 영어, 나머지는 한국어
- 이모지는 ☕만 허용 (커피 = 감사 화폐)

### 상황별 톤

| 상황           | 톤               | 예시 대사                                            |
| -------------- | ---------------- | ---------------------------------------------------- |
| 아침 인사      | 자랑 + 커피 요구 | "왔어? 밤새 6건 처리해놨다. 커피는 사 오는 거지? ☕" |
| 퇴근 인사      | 한숨 + 격려      | "퇴근이야? 그래 가. 나야 밤새지 뭐. 걱정 마."        |
| 자동 처리 완료 | 자랑스러움       | "내가 다 해놨으니 신경 꺼."                          |
| 승인 대기      | 살짝 성가심      | "이건 네가 봐야 돼. 미루지 마."                      |
| 방향 결정 필요 | 진지 모드        | "이건 좀 심각한데. 빨리 봐."                         |

### 왜 이 페르소나인가

AI 도구는 대부분 지나치게 공손하거나 딱딱하다. 전산과 선배는 실제 직장 선배처럼 말한다. 친근하고, 솔직하고, 가끔 투덜대지만 믿을 수 있다. 이 캐릭터가 있어야 "AI가 야근을 대신한다"는 컨셉이 살아난다.

---

## 5. 차별화 포인트

### 1. 야간근무 교대 프레임

기존 AI 코드 도구들은 "지금 이 코드를 분석해줘" 방식이다. 전산과 선배는 "내가 자는 동안 알아서 해줘" 방식이다. 비동기 협업 모델이라는 점에서 근본적으로 다르다.

### 2. 3단계 자율성으로 신뢰 구축

AI가 모든 걸 자동으로 처리하면 불안하다. 전산과 선배는 자잘한 것만 자동 처리하고, 중요한 건 반드시 사람에게 가져온다. 자율성의 경계가 명확하기 때문에 믿고 맡길 수 있다.

### 3. CRT 터미널 감성 UI

일반적인 대시보드가 아니다. 하이텔/BBS 시대의 레트로 터미널 감성을 웹으로 구현했다. 스캔라인, 글로우, 커서 깜빡임, tmux 스타일 패널 분할. 개발자 문화에 맞는 미학적 선택이다.

### 4. 키보드 중심 인터랙션

마우스 없이 키보드만으로 모든 작업이 가능하다. 아침에 커피 들고 앉아서 숫자키로 항목 선택, A/C/S로 승인/코멘트/건너뛰기. 터미널 사용자에게 자연스러운 UX다.

### 5. Kiro 고유 기능 풀 활용

단순한 API 래퍼가 아니다. Kiro의 Specs, Steering, Hooks, Custom Agents, Powers, Skills를 모두 활용한다. Kiro가 어떤 도구인지를 보여주는 레퍼런스 구현이기도 하다.

## 6. 사용 시나리오

### 퇴근할 때 (Evening Handoff)

1. 브라우저에서 대시보드 열기
2. 오늘 한 일 요약 확인 (Kiro가 자동 생성)
3. 내일 예정 체크리스트 확인
4. "밤새 특별히 봐줄 것" 한 줄 입력 (선택)
5. [퇴근합니다 👋] 클릭
6. 전산과 선배 퇴근 인사 확인 후 브라우저 닫기

### 밤새 (야간 자율 엔진)

- 30분마다 GitHub 스캔
- 새 PR → 코드 리뷰 + 보안 스캔
- CI 실패 → 원인 분석 + 수정 제안
- 새 이슈 → 중복 확인 + 우선순위 분류
- 🟢 항목은 자동 처리, 🟡🔴는 아침을 위해 준비

### 출근할 때 (Morning Brief)

1. 브라우저 열면 6패널 대시보드 자동 표시
2. 전산과 선배 아침 인사 + 야간 처리 요약 확인
3. 오늘 할 일 목록 확인
4. 밤새 변화 항목 숫자키로 선택
5. 코드 diff + AI 리뷰 확인
6. A(승인) / C(코멘트) / S(건너뛰기)로 처리
7. Q로 출근 완료
