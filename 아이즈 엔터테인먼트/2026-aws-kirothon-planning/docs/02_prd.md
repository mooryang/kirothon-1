### 1.2 상태 머신

서비스는 세 가지 상태를 순환한다.

```
idle (대기)
  │
  ├─ [퇴근합니다] 클릭 ──→ night (야간 모니터링)
  │                              │
  │                              ├─ 30분마다 스캔 반복
  │                              │
  └─ [출근 완료] 클릭 ←── morning (아침 브리핑)
       (Q키)                     │
                                 └─ [출근 대시보드 열기] ──→ morning
```

- `idle`: 초기 상태. Evening Handoff 화면 표시.
- `night`: 야간 cron 실행 중. 모니터링 상태 화면 표시.
- `morning`: 야간 분석 완료. Morning Brief 대시보드 표시.

#### UX 흐름

1. 화면 진입 시 Kiro가 오늘 업무 요약을 자동 생성 (로딩 스피너 표시)
2. 오늘 한 일 패널: 커밋 수, PR 리뷰 수, 이슈 처리 수, 스프린트 진행률 표시
3. 내일 예정 패널: GitHub 마일스톤 + 대기 중인 PR 기반으로 자동 생성. 체크박스 토글 가능.
4. 특별 지시 입력: 터미널 스타일 텍스트 입력. 선택 사항.
5. [퇴근합니다] 버튼 클릭 → 퇴근 처리 시퀀스 시작

#### 퇴근 처리 시퀀스

```
[퇴근합니다] 클릭
  │
  ├─ 1. POST /api/clock-out { instruction: "..." }
  │     └─ sessions 테이블에 퇴근 기록 저장
  │
  ├─ 2. GitHub baseline 스냅샷 수집
  │     └─ 현재 PR, 이슈, CI, 커밋 상태 저장
  │
  ├─ 3. 30분 간격 cron 등록
  │
  └─ 4. 야간 전환 브릿지 화면으로 전환
        ├─ 선배 대사: "수고했다. 맡겨만 둬."
        ├─ 터미널 로그: "baseline 수집 중... 완료"
        └─ 2초 후 → 야간 모니터링 상태 화면
```

### 2.3 Detail View (항목 상세)

#### 화면 목적

밤새 변화 항목 하나를 선택했을 때 코드 diff와 AI 리뷰를 함께 보는 화면.

│  @@ -45,7 +45,12 @@          │                              │
│  -  if (token.exp < now) {   │  리스크: MEDIUM              │

#### 인터랙션

| 키  | 동작                                                        |
| --- | ----------------------------------------------------------- |
| A   | 승인 → /api/action {action: "approve"} → Morning Brief 복귀 |
| C   | 코멘트 입력 모달 → /api/action {action: "comment"} → 복귀   |
| S   | 건너뛰기 → 다음 항목 or Morning Brief                       |
| B   | Morning Brief로 돌아가기                                    |

#### diff 색상 규칙

- `+` 로 시작하는 줄: 터미널 그린 `#00ff41`, 배경 `rgba(0,255,65,0.1)`
- `-` 로 시작하는 줄: 터미널 레드 `#ff3333`, 배경 `rgba(255,51,51,0.1)`
- 나머지: 회색 `#666666`

### 2.4 타임랩스 모드 (데모 전용)

#### 목적

발표/시연 시 밤새 AI가 일하는 과정을 빠르게 재생.

#### 접근 방법

URL에 `?mode=timelapse` 파라미터 추가.

#### 동작 흐름

1. 시드 데이터에서 시간순 이벤트 목록 로드
2. 500ms 간격으로 이벤트 하나씩 재생
3. 야간 로그 패널에 메시지 추가 (slideUp 애니메이션)
4. recharts 차트 실시간 업데이트
5. 처리 현황 카운터 업데이트 (🟢 N건 처리)
6. 전산과 선배 대사 중간중간 삽입 (typing-effect)

## 3. 야간 자율 엔진 상세

### 3.1 스캔 사이클 흐름

```
scheduler (30분 간격 cron)
  │
  ├─ 1. scanner.js: GitHub 현재 상태 수집
  │     └─ PR, 이슈, CI, 커밋 데이터 수집
  │
  ├─ 2. differ.js: baseline 대비 diff 추출
  │     └─ 새 PR, 새 이슈, CI 상태 변화, 새 코멘트 감지
  │
  ├─ 3. analyzer.js: night-summarizer 에이전트 호출
  │     └─ night-summarizer가 내부적으로 하위 에이전트에 위임:
  │         ├─ #pr-reviewer: PR별 코드 리뷰
  │         ├─ #ci-analyzer: CI 실패 분석
  │         ├─ #issue-triager: 이슈 분류/중복 감지
  │         └─ #security-scanner: 보안 취약점 스캔
  │
  ├─ 4. Hook(post-analysis): 분석 결과 자동 처리
  │     ├─ 🟢 auto → executor.js로 자동 처리
  │     └─ 🟡🔴 → DB 저장만
  │
  └─ 5. 야간 로그 기록 → 다음 사이클 대기
```

### 3.2 Kiro 에이전트 역할 분담

| 에이전트         | 역할                                           | 입력                       | 출력                |
| ---------------- | ---------------------------------------------- | -------------------------- | ------------------- |
| night-summarizer | 오케스트레이터. 하위 에이전트 위임 + 결과 취합 | 전체 diff 목록             | 종합 분석 결과 JSON |
| pr-reviewer      | PR 코드 리뷰. 리스크 평가.                     | PR diff + 파일 목록        | 리뷰 결과 JSON      |
| ci-analyzer      | CI 실패 원인 분석. 관련 PR 연관성 파악.        | CI 로그                    | 분석 결과 JSON      |
| issue-triager    | 이슈 분류. 중복 감지. 우선순위 평가.           | 이슈 내용 + 기존 이슈 목록 | 분류 결과 JSON      |
| security-scanner | OWASP 기준 보안 취약점 전문 스캔.              | PR diff                    | 보안 스캔 결과 JSON |

### 3.3 자동 처리 안전장치

🟢 auto로 분류되어도 아래 조건을 모두 만족해야만 자동 처리:

**PR 자동 머지 조건**

- dependabot이 생성한 PR일 것
- 모든 CI 체크가 통과 상태일 것
- main 브랜치 직접 머지가 아닐 것

**이슈 자동 클로즈 조건**

- 중복 이슈로 확인된 것일 것
- 원본 이슈 번호가 명시되어 있을 것

## 4. 데이터 흐름

### 4.1 퇴근 시 데이터 흐름

```
사용자 입력 (instruction)
  │
  ▼
POST /api/clock-out
  │
  ├─ sessions 테이블 INSERT (type: clock_out, instruction)
  │
  ├─ scanner.collectBaseline()
  │   └─ snapshots 테이블 INSERT (prs, issues, ci, commits)
  │
  └─ scheduler.start() → 30분 cron 등록
```

### 4.2 야간 스캔 시 데이터 흐름

```
cron 트리거 (30분마다)
  │
  ├─ scanner.collectCurrent() → 현재 상태 수집
  │
  ├─ differ.getDiff(baseline, current)
  │   └─ diffs 테이블 INSERT (type, ref_id, title, data)
  │
  ├─ analyzer.analyzeAllDiffs(diffs)
  │   └─ Kiro CLI 호출 → analyses 테이블 INSERT
  │       (category, summary, detail, risk_level, suggestion)
  │
  └─ executor.processAuto(analyses)
      └─ actions 테이블 INSERT (action_type, status, result)
```

### 4.3 출근 시 데이터 흐름

```
POST /api/clock-in
  │
  ├─ sessions 테이블 INSERT (type: clock_in)
  │
  ├─ night-summarizer 에이전트 호출
  │   └─ daily_summaries 테이블 INSERT
  │
  └─ GET /api/status → morning 상태 반환
      └─ MorningBrief 컴포넌트에 데이터 전달
```

### 5.1 CRT 터미널 테마

**색상 팔레트**

| 변수            | 값        | 용도                     |
| --------------- | --------- | ------------------------ |
| `--term-green`  | `#00ff41` | 기본 텍스트, 🟢 항목     |
| `--term-amber`  | `#ffb000` | 패널 레이블, 강조 텍스트 |
| `--term-cyan`   | `#00d4ff` | 보조 강조, 링크          |
| `--term-red`    | `#ff3333` | 🔴 항목, diff 삭제 줄    |
| `--term-yellow` | `#ffff00` | 🟡 항목                  |
| `--term-bg`     | `#0a0a0a` | 배경                     |
| `--term-border` | `#333333` | 패널 테두리              |

**CRT 효과 레이어**

1. 스캔라인: `repeating-linear-gradient` 오버레이 (z-index 9999)
2. 비네팅: `box-shadow inset` 으로 화면 가장자리 어둡게
3. 텍스트 글로우: `text-shadow 0 0 5px rgba(0,255,65,0.4)`
4. 커서 깜빡임: `blink 1s step-end infinite` 애니메이션

**예외: 차트 패널**
recharts 차트가 있는 패널에는 스캔라인 오버레이 미적용 (`no-scanline` 클래스).

### 5.2 타이포그래피

- 폰트: JetBrains Mono (woff2, 로컬 호스팅)
